/// Quickstart

From Zero to Deployed in 5 Commands

Everything you need to go from idea to production.

1

Initialize your project

lpc init my-app

Scaffolds a new project with config, git, and folder structure. Supports templates: --template saas, --template api, --template automation.

2

Install packs

lpc install @lp/auth @lp/payments

Pulls verified packs from the marketplace. Auto-resolves dependencies, checks contracts, and wires into your project.

3

Start the dev server

lpc dev

Launches backend + frontend with hot reload, self-healing loops, and telemetry dashboard at localhost:9000.

4

Test everything

lpc test

Runs contract tests across all installed packs, checks for breaking changes, and verifies integration health.

5

Deploy to production

lpc deploy --prod

Ships to LP Cloud or your own infra. Zero-downtime deploys with rollback, monitoring, and self-healing enabled.

~/my-app — deploy
$ lpc deploy --prod ◆ Building production bundle... ✓ Backend compiled (FastAPI) ✓ Frontend compiled (React) ✓ Assets optimized — 2.4 MB ◆ Running contract tests... ✓ @lp/auth — 14/14 passed ✓ @lp/payments — 9/9 passed ✓ @lp/dashboard — 22/22 passed ◆ Deploying to LP Cloud... ✓ Containers pushed ✓ DNS configured ✓ SSL provisioned ✓ Self-healing enabled ◆ Deployment complete! → https://my-app.lp.app deployed in 34s

/// Why LPC

Built for Speed

Instant Installs

Parallel resolution, local cache, and deduplication. Install 10 packs in under 2 seconds.

Contract Verification

Every pack has typed contracts. LPC verifies compatibility before install — no runtime surprises.

Self-Healing

lpc heal detects failing tests, analyzes the root cause, and auto-patches your code.

Dev Server

One command starts backend + frontend + telemetry dashboard. Hot reload across the full stack.

One-Command Deploy

Deploy to LP Cloud or self-host. Zero-downtime, auto-rollback, SSL, and monitoring — all built in.

LPDN Support

Push apps to LPDN nodes for offline distribution. lpc lpdn push handles everything.
